# 全美股日内交易策略框架 v2
# OpenAI + Alpaca | PDT限制下的精选交易系统

---

## 一、核心理念

**每周只有3颗子弹，必须打最值钱的靶子。**

你面对的是4000+只美股，但PDT规则限制你5个交易日内最多3次日内交易。这意味着系统的核心任务不是"发现信号"——信号每天有几十个——而是**排序和筛选**，从几十个机会中挑出本周最值得用子弹的3个。

LLM的角色：不预测价格，而是做全市场信息处理器。帮你在几分钟内读完人类需要几小时才能处理的新闻和数据，然后排出优先级。

## 三、第一层：全市场技术面扫描器（零API成本）

### 目的

从4000+只股票中快速筛出"今天有异动"的30-80只。纯本地计算，不花一分钱API。

### 扫描时间点

| 时间 | 扫描内容 |
|------|----------|
| 盘前 4:00-9:30 AM ET | 盘前价格异动、跳空缺口、盘前成交量 |
| 开盘 9:30-9:45 AM ET | 开盘15分钟放量、价格突破 |
| 盘中 每15分钟 | 持续监控已筛出的标的 + 新异动检测 |

### 筛选条件（满足任一即入选）

```
1. 异常放量
   当前成交量 > 前20日同时段均量 × 3

2. RSI极端值
   RSI(14) < 25 或 RSI(14) > 75（比常规30/70更严格，减少候选数）

3. 价格跳空
   开盘价 vs 昨收 > ±3%

4. 盘前异动
   盘前成交量 > 10万股 且 盘前涨跌幅 > ±2%

5. 布林带突破
   价格突破布林带(20,2) 上轨或下轨

6. 日内动量
   过去30分钟涨跌幅 > ±2% 且 成交量放大

7. VWAP偏离
   当前价格偏离VWAP > ±2%
```

### 预过滤（避免垃圾股）

扫描前先排除：
- 股价 < $5（低价股波动大但滑点更大，不适合小账户）
- 日均成交量 < 50万股（流动性不够，进出困难）
- 市值 < $3亿（太小的公司新闻覆盖差，AI分析无数据可用）

### 输出

每只通过筛选的股票输出一个结构化数据包：
```json
{
  "ticker": "AAPL",
  "trigger_type": "volume_spike",
  "current_price": 185.30,
  "change_pct": 3.2,
  "volume_ratio": 4.5,
  "rsi": 78,
  "vwap": 182.10,
  "bollinger_position": "above_upper",
  "avg_spread": 0.02,
  "market_cap": 2850000000000,
  "sector": "Technology"
}
```

## 四、第二层：新闻情绪快筛（gpt-4.1-mini）

### 目的

对第一层筛出的30-80只股票查新闻、做情绪分析，筛掉"技术异动但没有基本面支撑"的噪音。

### 双向验证逻辑

```
情况A：技术异动 + 有匹配新闻 → 信号增强，进入第三层
情况B：技术异动 + 无新闻       → 可能是内幕/机构行为，保留但降权
情况C：重大新闻 + 技术未反应   → 市场还没price in，高优先级候选
情况D：无异动 + 无新闻         → 跳过（第一层已过滤）
```

**情况C是最有价值的。** 新闻出来但股价还没动，说明你有时间窗口。

### 新闻来源

- **主力**：Alpaca News API（按ticker过滤，有唯一ID可去重）
- **补充**：OpenAI Web Search（每小时对第一层筛出的股票搜一次突发新闻）

### 模型调用

| 场景 | 模型 | 原因 |
|------|------|------|
| 常规新闻情绪分类 | gpt-4.1-mini | 速度快，情绪分类够用 |
| 置信度 < 50的复杂新闻 | gpt-5.4 | 需要理解二阶效应 |
| 宏观政策/地缘新闻 | gpt-5.4 | 需要判断对不同行业的差异化影响 |

### Prompt（gpt-4.1-mini）

```
你是{TICKER}（{公司全名}，{行业}）的专属日内交易新闻分析师。

只评估这条新闻在今天交易日内的影响，不关心长期影响。

新闻：{news_text}
当前技术面：价格{price}，涨跌{change}%，成交量比{vol_ratio}x

返回JSON：
{
  "sentiment": "bullish" | "bearish" | "neutral",
  "confidence": 0-100,
  "intraday_severity": 1-10,
  "catalyst_type": "earnings" | "guidance" | "analyst" | "macro" | "sector" | "legal" | "product" | "insider" | "other",
  "expected_move_pct": 估计今日可能波动幅度,
  "summary": "一句话"
}

如果不确定，confidence设为50以下。
```

### 筛选标准

进入第三层的条件：
- sentiment 为 bullish 或 bearish（中性跳过）
- confidence ≥ 60
- intraday_severity ≥ 5
- 或者：技术异动非常强（volume_ratio > 5），即使没新闻也保留

## 五、第三层：深度排名（o3）

### 目的

这是整个系统最关键的一步。从10-20只候选中选出**本周最值得用日内交易名额的Top 3**。

### 为什么用o3而不是其他模型

o3的多步推理能力在这里至关重要——它不是在分析单只股票，而是在做**横向比较**：
- A股票新闻更猛但技术面已经price in了
- B股票信号弱一点但风险回报比更好
- C股票各方面都不错但今天流动性不够

这种权衡多个维度的排序任务，o3比其他模型有明显优势。

### 调用时机

- **盘前研判（主力）**：9:00-9:25 AM ET，把过夜新闻+盘前数据汇总，排出今日Top 3候选
- **盘中更新（补充）**：如果盘中出现新的强信号（比如某公司突发重大新闻），重新排名

### Prompt

```
你是一个日内交易基金的投资组合经理。

约束条件：
- 本周剩余日内交易次数：{remaining_day_trades}（5个交易日滚动窗口内最多3次）
- 账户总值：${portfolio_value}
- 单笔最大仓位：账户的15%
- 今天是{weekday}，本周还剩{remaining_days}个交易日

以下是今天通过两轮筛选的候选股票，每只附带技术面数据和新闻分析：

{candidates_json}

请做以下工作：

1. 对每只候选股票评估：
   - 日内交易的预期收益率（考虑方向和幅度）
   - 风险（可能的最大亏损）
   - 风险回报比
   - 信号时效性（还有多少时间窗口）
   - 流动性（能否顺利进出）

2. 横向对比所有候选，排出优先级

3. 考虑"子弹分配策略"：
   - 如果今天是周一且剩余3次，可以用1次
   - 如果今天是周四且剩余1次，必须极度挑剔
   - 如果今天所有候选都不够强，建议保留名额

4. 输出最终推荐

展示你的完整思考过程。

返回JSON：
{
  "analysis_date": "2026-03-27",
  "remaining_trades": 3,
  "recommendations": [
    {
      "rank": 1,
      "ticker": "AAPL",
      "action": "BUY" | "SHORT",
      "confidence": 0-100,
      "entry_price": 185.50,
      "stop_loss": 183.00,
      "take_profit": 190.00,
      "position_size_pct": 10,
      "risk_reward_ratio": 1.8,
      "time_window": "9:30-11:00 AM",
      "reasoning": "完整推理"
    }
  ],
  "save_bullets": true | false,
  "save_reason": "如果建议不交易，解释为什么",
  "market_context": "今天大盘环境概述",
  "risk_warnings": ["全局风险因素"]
}
```

### 关键设计：子弹分配策略

o3不只是选股，还要帮你**管理稀缺资源（3次名额）的分配**。

```
周一：剩余3次 → 标准严格度，好机会就用
周二：剩余2-3次 → 标准严格度
周三：剩余1-2次 → 提高标准，只打A+级机会
周四：剩余1次 → 极度挑剔，或者建议保留到周五
周五：剩余1-3次 → 名额即将重置，可以稍微放宽

如果本周已用完3次 → 只监控不交易，记录错过的机会用于复盘
```

## 六、第四层：执行与PDT管理（Alpaca API）

### PDT追踪器（硬编码，不可被AI覆盖）

```python
# 系统必须在每次下单前检查
class PDTTracker:
    """
    维护滚动5个交易日窗口内的日内交易计数。
    任何情况下不允许第4次日内交易。
    """
    MAX_DAY_TRADES = 3
    WINDOW_DAYS = 5  # 滚动交易日

    def can_day_trade(self) -> bool:
        """检查是否还有日内交易名额"""
        recent_trades = self.get_trades_in_window()
        return len(recent_trades) < self.MAX_DAY_TRADES

    def remaining_trades(self) -> int:
        """返回剩余可用日内交易次数"""
        return self.MAX_DAY_TRADES - len(self.get_trades_in_window())
```

**铁律：如果 `remaining_trades() == 0`，系统直接静默，不调任何API，不分析任何股票。省钱+避免手痒。**

### 下单逻辑

```
1. 检查PDT名额 → 不够直接跳过
2. 检查风险管理规则 → 不符合直接跳过
3. 下单（Limit Order优先，不用Market Order）
4. 设置止损单（bracket order）
5. 设置收盘前自动平仓（日内交易必须当天了结）

关键：必须在收盘前平仓！
  - 3:45 PM ET 开始检查所有日内持仓
  - 3:50 PM ET 未平仓的自动市价平仓
  - 这是硬规则，否则变成隔夜持仓，不算日内交易但承担隔夜风险
```

### 订单类型

```
入场：Limit Order（比现价差0.05-0.10，避免追高）
止损：Stop-Limit Order（止损价 = ATR × 1.5）
止盈：Limit Order（目标价 = 入场价 + ATR × 2-3）
尾盘平仓：Market Order（3:50 PM ET强制执行）
```

## 七、风险管理规则（硬编码，AI不可覆盖）

### 仓位规则

| 规则 | 限制 |
|------|------|
| 单笔最大仓位 | 账户总值的 **15%** |
| 同时持仓数 | 最多 **2只**（日内交易需要集中注意力） |
| 单笔最大亏损 | 账户总值的 **1.5%**（小账户必须更保守） |
| 单日最大亏损 | 账户总值的 **3%**（触发后当日停止交易） |

### 为什么比之前更保守

之前设的是单笔2%、单日5%——那是基于6只股票Swing交易的参数。日内交易全市场，波动更大、不确定性更高，而且你账户$5K-$25K经不起连续重击。1.5%单笔亏损意味着$10,000的账户单笔最多亏$150，这是可控的。

### 冷却规则

- 连续2笔亏损 → 当日停止交易
- 单周亏损 > 账户5% → 下周减半仓位上限
- 连续2周亏损 → 系统暂停1周，只监控不交易，用这周做策略复盘

## 九、一天的完整工作流

### 盘前 (4:00 - 9:30 AM ET)

```
4:00 AM   系统启动，检查PDT剩余名额
          → 0次剩余？系统静默，今天不交易

4:05 AM   拉取所有股票的盘前数据
          → 本地扫描盘前异动（跳空、放量）
          → 筛出盘前候选 20-50只

4:10 AM   对候选股票拉取过夜新闻
          → gpt-4.1-mini 情绪分类
          → 复杂新闻 escalate 到 gpt-5.4

8:30 AM   OpenAI Web Search 补充搜索
          → 抓取可能遗漏的突发新闻

9:00 AM   汇总所有数据，调 o3 做盘前排名
          → 输出今日 Top 3 候选 + 子弹分配建议
          → 设定每只候选的入场价、止损、止盈

9:25 AM   人工最终确认（可选）
          → 系统发通知：今日推荐 / 建议保留名额
```

### 盘中 (9:30 AM - 3:45 PM ET)

```
9:30      市场开盘
          → 监控Top 3候选的价格行为
          → 到达入场价 → 下Limit Order

9:30-10:30 开盘30-60分钟是波动最大的时段
          → 持续监控新闻流
          → 如有突发重大新闻 → 重新触发第二层+第三层

10:30+    波动减小
          → 降低扫描频率（每30分钟一次）
          → 监控已持仓的止损/止盈

每15分钟  新闻轮询
          → 新的重大新闻 → mini分析 → 如果改变局面 → o3重新评估

实时      价格监控
          → 触发止损 → 自动平仓
          → 触发止盈 → 自动平仓
          → 移动止损：盈利 > 1% 后，止损上移到保本价
```

### 尾盘 (3:45 - 4:00 PM ET)

```
3:45 PM   检查所有日内持仓
          → 未平仓 → 3:50 PM 强制市价平仓
          → 不允许日内持仓隔夜

4:00 PM   收盘
          → 记录今日所有交易结果
          → 更新PDT计数器
          → 生成日报（盈亏、信号准确率、模型成本）
```

### 盘后 (4:00 PM - 8:00 PM ET)

```
4:00-6:00 PM  盘后新闻扫描（轻量级）
              → 只看重大新闻，为明天做准备
              → 财报发布（很多公司盘后发财报）
              → o3 分析重大财报 → 标记为明日候选

6:00 PM       生成每日复盘报告
              → 今日信号 vs 实际结果
              → AI推荐 vs 实际价格走势
              → 用于长期策略优化
```

## 十、PDT名额用完后的应对方案

5个交易日内3次用完后，系统不是关机，而是切换模式：

### 只监控不交易

- 继续跑第一层和第二层扫描
- 记录"如果能交易，我会交易什么"
- 这些虚拟交易用于回测和优化策略

### 为什么这很重要

你交易名额恢复后（滚动窗口内最早的那笔日内交易满5天后），系统已经积累了几天的监控数据和虚拟信号，可以立刻以最佳状态恢复交易。

## 十三、关键提醒

1. **PDT是铁律。** 系统层面硬锁3次/5天，任何情况不可突破。第4次日内交易 = 账户冻结90天。

2. **子弹比信号珍贵。** 不是每个好信号都值得用名额。周一用了2次，周四出现绝佳机会但没子弹了——这才是最大的风险。o3的"子弹分配"功能是整个系统的灵魂。

3. **日内交易的时间窗口。** 开盘后30-60分钟波动最大、机会最多，也是最容易犯错的。盘中11:00-2:00通常是死水期，不值得浪费子弹。2:30-3:30是第二波活跃期。

4. **尾盘平仓不可协商。** 3:50 PM没平仓就强制市价平。隔夜风险对小账户是致命的——一条盘后新闻可以让股价第二天跳空10%。

5. **你的edge不是速度。** 高频交易机构比你快1000倍，不要试图抢毫秒。你的edge是AI帮你处理信息的广度——它能同时读4000只股票的新闻，而机构的分析师一个人只能盯10只。

6. **$6000 API预算用不完。** 不要用"省API钱"作为降低分析质量的理由。该用o3的时候就用，该调gpt-5.4的时候就调。你的瓶颈是策略质量和数据质量，不是API成本。
